# v1.1.0 更新规划

## 目标

v1.1.0 聚焦基础修复与补齐，在不扩大业务边界的前提下，把当前 v1.0.1 的静态接口原型整理成更稳定、可验证、可继续迭代的版本。

## 基线

- 当前真实基线按 Git tag 认定为 `v1.0.1`。
- `v1.0.1` 已将前端数据读取切换到 `/api/*.json` 静态接口。
- README 中仍存在 `v1.0.0` 口径，本次同步修正。

## 实施范围

- 文档补齐：合入 5-10 分钟讲解提纲，更新 README 和版本迭代事项。
- 数据治理：以 `data/*.csv` 作为源数据，通过脚本生成 `api/*.json`。
- KPI 修复：Dashboard 同时展示 `P0 紧急` 和 `超时事件` 两个指标。
- 事件流转补强：事件详情的状态流程栏明确标注当前节点。
- AI 入口补强：事件详情提供对应事件的 AI 辅助入口，并可跳转定位到 AI 建议卡。
- 验证补齐：重写验证脚本，新增测试目录，覆盖数据结构、关联关系、KPI 计算和模块入口。
- 编码修复：所有 `api/*.json` 输出为 UTF-8 without BOM。

## 数据口径

- `data/events.csv` 新增 `isOverdue` 字段，作为真实超时口径。
- `urgent` 统计未关闭且 `priority = P0` 的事件数。
- `overdue` 统计未关闭且 `isOverdue = true` 的事件数。
- 片区 `todayEvents`、`pendingEvents`、`overdueEvents` 由事件数据生成，避免手工维护重复统计。

## 验收命令

```powershell
powershell -ExecutionPolicy Bypass -File scripts/sync-api-from-csv.ps1 -Check
powershell -ExecutionPolicy Bypass -File scripts/verify-prototype-base.ps1
python -m unittest discover -s tests
```

## 边界

- v1.1.0 不接外部 AI API，不新增写接口，不做状态持久化。
- 事件详情的 AI 入口使用现有 `api/aiSuggestions.json` 静态建议数据定位，不调用外部模型。
- 外部 AI API 和提示词模板继续放在 v1.2.0。
- 事件流转、AI 建议采纳等写操作持久化继续放在 v1.3.0。
