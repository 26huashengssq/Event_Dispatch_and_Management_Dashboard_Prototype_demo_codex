## 4. 代码质量问题

| 编号 | 问题 | 位置 | 严重程度 |
|---|---|---|---|
| Q1 | **数据双重冗余**：`assets/js/data.js` 硬编码 + `api/*.json` 静态文件含相同数据；`data/*.csv` 完全未被使用 | `data.js` + `api/` + `data/` | ✅ v1.1.0 修复：CSV 生成 API JSON，删除 `data.js` |
| Q2 | API 响应带 UTF-8 BOM (`﻿`)，导致 PowerShell JSON 解析失败 | `api/*.json` 文件编码 | ✅ v1.1.0 修复：同步脚本写入 UTF-8 without BOM |
| Q3 | 验证脚本 `verify-prototype-base.ps1` 检查的 `data-render` 等标记属于旧静态架构，8/9 全部不匹配 | `scripts/` | ✅ v1.1.0 修复：重写为 Tab + ES Module + API 架构检查 |
| Q4 | KPI `overdue` 变量名与含义不一致（计算"P0 未关闭数"非"超时数"） | `api/kpi.json` | ✅ v1.1.0 修复：拆分 `urgent` 和 `overdue` |
| Q5 | 前端多一层不必要的网络调用（API 层从本地 server 拉取 → 实际数据来源于同仓库静态文件） | `api.js` | 低 |
| Q6 | Worktree 测试文件（3 个）未合入主分支 | `.worktrees/` | ✅ v1.1.0 修复：迁移测试意图到 `tests/` |

## 7. 优先级路径

```
v1.1.0 (当前冲刺)              v1.2.0 (关键缺口)            v1.3.0              v2.0.0
┌──────────────────────┐  ┌─────────────────────────┐  ┌──────────────┐  ┌──────────────┐
│ KPI修复 · 数据去重    │  │ 提示词模板 · API骨架复用  │  │ 事件写接口    │  │ 权限/报表     │
│ 讲解文档 · 验证脚本    │→ │ POST /api/ai/generate    │→ │ 建议写接口    │→ │ 实时推送      │
│ 合入测试 · BOM修复    │  │ 前端实时接入 · 环境变量   │  │ KPI动态更新   │  │ 真实数据库    │
└──────────────────────┘  └─────────────────────────┘  └──────────────┘  └──────────────┘
```

> **核心结论**：需求第四条明确要求 AI 接入外部 API + 提示词设计，这是 v1.0.0 最关键的缺失。`feature-ai-assist` worktree 已有 API 接入骨架，v1.2.0 应优先复用。
