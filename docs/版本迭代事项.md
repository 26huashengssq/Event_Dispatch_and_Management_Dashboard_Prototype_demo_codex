# 版本迭代事项文档

> 基于 `requirement.md` 需求文档，对照当前版本 (v1.0.0) 逐功能点检查，梳理差距并规划后续迭代。

---

## 1. 需求对照检查

### 1.1 需求一：让管理层快速看清当前各片区运行状态

| 检查项 | 状态 | 说明 |
|---|---|---|
| 片区状态一览（正常/预警/异常/处理中） | ✅ 已实现 | 4 个片区卡片 + 颜色标签 |
| 异常事件数量 | ✅ 已实现 | `todayEvents` / `pendingEvents` / `overdueEvents` |
| 待处理事件数量 | ✅ 已实现 | KPI 栏展示"待处理" |
| 超时提醒 | ⚠️ 部分实现 | KPI 栏"P0 紧急"实际计算的是 P0 且未关闭的事件数，命名与含义有偏差 |
| 关键提醒 / 异常高亮 | ⚠️ 可增强 | 缺少醒目的顶部告警横幅，仅靠卡片颜色区分 |

### 1.2 需求二：让管理人员知道事件卡在哪、谁负责、下一步该做什么

| 检查项 | 状态 | 说明 |
|---|---|---|
| 事件流转阶段可视化 | ✅ 已实现 | 5 阶段节点（待派单→协调中→处理中→待复核→已关闭） |
| 点击阶段筛选事件 | ✅ 已实现 | 有效且可取消筛选 |
| 事件详情面板（位置/负责人/状态/时限） | ✅ 已实现 | 详情面板完整展示所有字段 |
| 下一步动作明确展示 | ✅ 已实现 | `nextAction` 字段 + 操作面板"推荐动作" |
| 操作反馈（确认推进/改派/升级） | ✅ 已实现 | 三个按钮 + 反馈消息 |
| 操作后状态持久化 | ❌ 未实现 | 当前为纯前端模拟，刷新后丢失 |

### 1.3 需求三：体现 AI 辅助判断或调度（需接入外部 API，有提示词）

| 检查项 | 状态 | 说明 |
|---|---|---|
| AI 风险态势分析 | ✅ 已实现 | 四象限风险卡片 |
| 调度建议卡片（推荐动作/原因/资源） | ✅ 已实现 | 4 条建议各有置信度进度条 |
| 采纳/调整/暂存操作 | ✅ 已实现 | 前端交互完整 |
| 接入外部 AI API | ❌ 未实现 | 当前为静态模拟数据，无真实 API 调用 |
| 片区-事件维度的提示词设计 | ❌ 未实现 | 需求明确标注"针对片区-事件，需要有提示词" |
| 后端 AI 建议生成接口 | ❌ 未实现 | `server.js` 仅返回硬编码 JSON |

### 1.4 需求四：最小可用版本 (MVP)

| 检查项 | 状态 | 说明 |
|---|---|---|
| 覆盖核心交互闭环 | ✅ 已实现 | 总览→下钻→决策→反馈路径完整 |
| 不追求大而全 | ✅ 符合 | 无权限/排班/统计报表/历史追溯等 |

## 3. 代码质量问题

| 问题 | 位置 | 严重程度 |
|---|---|---|
| 数据重复定义：`server.js` 与 `assets/js/data.js` 各自硬编码一份相同数据 | 全栈 | 中 |
| CSV 数据文件 (`data/*.csv`) 未被服务端读取使用，形同虚设 | `server.js` / `data/` | 中 |
| 验证脚本 `verify-prototype-base.ps1` 检查的 HTML 标记与当前集成架构不匹配（如 `data-render` 属性已在重构中移除） | `scripts/` | 低 |
| KPI 计算：`overdue` 变量实际计算的是 P0 未关闭事件数，并非真正超时 | `data.js` + `server.js` | 低 |
| 前端模块通过 API 拉取数据，但实际 `api.js` 请求仍走本地 mock server，mock 层与数据层冗余 | `api.js` / `server.js` | 低 |

## 4. 版本迭代规划

### v1.1.0 — 基础修复与补齐

| 编号 | 事项 | 类型 | 说明 |
|---|---|---|---|
| 4.1.1 | 修正 KPI "P0 紧急"含义 | 修复 | 改为"超时事件"或正确命名；KPI 计算逻辑与实际含义对齐 |
| 4.1.2 | 服务端从 CSV 读取数据 | 重构 | `server.js` 改为读取 `data/*.csv`，消除前后端数据重复定义 |
| 4.1.3 | 合入讲解说明文档 | 补齐 | 从 `feature/deliverables` worktree 提取 `presentation/5.讲解提纲.md` 并入 main |
| 4.1.4 | 更新验证脚本 | 修复 | `verify-prototype-base.ps1` 适配当前集成架构的标记与文件结构 |

### v1.2.0 — AI 外部 API 接入

| 编号 | 事项 | 类型 | 说明 |
|---|---|---|---|
| 4.2.1 | 设计 AI 提示词模板 | 新增 | 针对"片区-事件"维度，设计结构化提示词：片区状态 + 事件信息 + 资源可用性 → AI 输出风险等级/推荐动作/原因/资源 |
| 4.2.2 | 新增 `/api/ai/generate` 后端接口 | 新增 | 接收 `districtId` + `eventId`，拼接提示词，调用外部 AI API（如 OpenAI），返回结构化建议 |
| 4.2.3 | 前端接入实时 AI 建议 | 改造 | `aiassist.js` 增加"生成建议"按钮，调用新接口并展示结果；保留静态数据作为 fallback |
| 4.2.4 | 环境变量配置 API Key | 新增 | 支持通过 `OPENAI_API_KEY` 等环境变量配置外部 API 凭证 |

### v1.3.0 — 状态持久化

| 编号 | 事项 | 类型 | 说明 |
|---|---|---|---|
| 4.3.1 | 事件流转状态持久化 | 新增 | "确认推进/改派/升级"操作结果写入文件存储，刷新不丢失 |
| 4.3.2 | AI 建议采纳状态持久化 | 新增 | 采纳/暂存状态持久化 |
| 4.3.3 | 新增 `/api/events/:id/action` 写接口 | 新增 | 支持 POST 推进/改派/升级事件 |
| 4.3.4 | 新增 `/api/suggestions/:id/accept` 写接口 | 新增 | 支持 AI 建议采纳操作持久化 |

### v2.0.0 — 生产增强（远期）

| 编号 | 事项 | 类型 | 说明 |
|---|---|---|---|
| 4.4.1 | 角色权限控制 | 新增 | 高管层只读 vs 调度主管可操作 |
| 4.4.2 | 统计报表与历史追溯 | 新增 | 事件趋势图、片区对比、处理时效统计 |
| 4.4.3 | 实时推送 | 新增 | WebSocket/SSE 实现事件状态实时更新 |
| 4.4.4 | 真实数据库 | 重构 | 替换文件存储为 SQLite/PostgreSQL |
