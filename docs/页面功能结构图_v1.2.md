# 一页纸页面/功能结构图 v1.2.0

## 一、Tab 导航与页面路由

```mermaid
flowchart TD
    subgraph 浏览器入口
        A[localhost:4000/index.html]
    end

    A --> B[JS 主控 app.js]
    B --> C[renderTabs 渲染 3 个 Tab 按钮]

    C --> D["Tab 1: 📊 片区总览"]
    C --> E["Tab 2: 🔄 事件流转"]
    C --> F["Tab 3: 🤖 AI 辅助"]

    D --> D1[dashboard.js: initDashboard]
    E --> E1[eventflow.js: initEventFlow]
    F --> F1[aiassist.js: initAiAssist]

    E1 -->|"点击 查看 AI 建议 按钮"| NAV["dispatchEvent: app:navigate"]
    NAV -->|"tab=aiassist, eventId=..."| B
    B --> F1
```

**路由机制**：
- 单页应用，通过 `index.html` 中的 `#module-container` 动态渲染
- Tab 切换由 `app.js` 中 `currentTab` 状态驱动，映射到对应模块初始化函数
- 跨模块导航通过 `window.dispatchEvent(new CustomEvent("app:navigate", { detail: { tab, eventId } }))` 实现

## 二、数据架构

```mermaid
flowchart LR
    subgraph 数据源
        CSV1["data/districts.csv"]
        CSV2["data/events.csv"]
        CSV3["data/flowStages.csv"]
        CSV4["data/aiSuggestions.csv"]
    end

    subgraph 同步脚本
        SYNC["scripts/sync-api-from-csv.ps1"]
    end

    subgraph API数据层
        J1["api/districts.json"]
        J2["api/events.json"]
        J3["api/flowStages.json"]
        J4["api/aiSuggestions.json"]
        J5["api/kpi.json"]
    end

    subgraph 服务层
        SRV["server.js (Node.js http)"]
    end

    subgraph 前端
        FE["api.js: fetch 封装"]
        DASH["dashboard.js"]
        FLOW["eventflow.js"]
        AI["aiassist.js"]
    end

    CSV1 --> SYNC
    CSV2 --> SYNC
    CSV3 --> SYNC
    CSV4 --> SYNC

    SYNC --> J1
    SYNC --> J2
    SYNC --> J3
    SYNC --> J4
    SYNC --> J5

    J1 --> SRV
    J2 --> SRV
    J3 --> SRV
    J4 --> SRV
    J5 --> SRV

    SRV -->|"GET /api/* → 读取 *.json"| FE
    SRV -->|"POST /api/ai/generate → AI代理"| FE

    FE --> DASH
    FE --> FLOW
    FE --> AI
```

**数据流说明**：
- CSV 是源数据，由 PowerShell 脚本一次性同步为 `api/*.json`
- Node.js server 按路径映射：`/api/districts` → `api/districts.json`
- 前端 `api.js` 封装 `fetch`，所有模块通过统一函数获取数据
- KPI 聚合数据 (`api/kpi.json`) 由同步脚本从 events 和 districts 统计生成

## 三、AI API 调用链

```mermaid
flowchart TD
    BTN["用户点击 生成建议 按钮"] --> FE_CALL["aiassist.js: handleGenerate()"]
    FE_CALL --> API_JS["api.js: generateAiSuggestion(eventId)"]
    API_JS --> POST["POST /api/ai/generate\\nBody: { eventId }"]

    POST --> SRV_HANDLE["server.js: handleAiGenerate()"]
    SRV_HANDLE --> LOAD["loadAiContext(eventId)\\n读取片区 + 事件 + 静态建议"]
    LOAD --> PROMPT["buildPrompt(context)\\n组装提示词模板"]

    PROMPT --> CHECK{"OPENAI_API_KEY\\n是否配置?"}

    CHECK -->|"是"| AI_CALL["fetch(OPENAI_BASE_URL)\\nBearer Token 认证\\nmodel: AI_MODEL\\ntemperature: 0.3"]
    AI_CALL --> PARSE["parseModelJson()\\n解析 AI 返回的 JSON"]
    PARSE --> NORM["normalizeAiSuggestion()\\n统一输出格式"]
    NORM --> RES_API["返回: { ... source: 'api' }"]

    CHECK -->|"否"| FALLBACK["buildFallbackSuggestion()\\n降级为本地静态建议"]
    AI_CALL -.->|"网络错误/非JSON"| FALLBACK
    FALLBACK --> RES_FB["返回: { ... source: 'fallback', fallbackReason }"]

    RES_API --> FE_RENDER["前端渲染建议卡片\\n显示 source 标签"]
    RES_FB --> FE_RENDER
```

**降级策略**：无 Key、网络异常、模型返回非 JSON 三种情况均自动 fallback，前端显示"fallback"标签和降级原因文字，不白屏。

## 四、组件树（按 Tab 展开）

### Tab 1: 📊 片区总览

```text
#module-container
└── panel-wide
    ├── .kpi-bar
    │   ├── .kpi-item × 5（事件总数 / 待处理 / P0紧急 / 超时 / 异常片区）
    │
    ├── section#district-overview
    │   ├── .panel-heading（标题 + 返回总览按钮）
    │   └── .district-grid
    │       └── .district-card × 4
    │           ├── .card-topline（片区名 + 状态标签）
    │           ├── .metric-row（今日事件 / 待处理 / 超时）
    │           └── .meta（可用资源）
    │
    └── section#event-list
        ├── .panel-heading（片区事件列表 / 全部事件列表）
        └── .event-stack
            └── .event-card × N（可筛选）
                ├── .card-topline（事件类型 + 优先级标签 P0/P1/P2）
                ├── .event-meta（状态标签 + 负责人）

```

## 五、角色视角对应关系

| 页面 / 功能 | 主要服务角色 | 页面目标 |
|---|---|---|
| Tab 1 片区总览 | 高管层 | 快速判断整体运行状态，发现异常片区 |
| Tab 2 事件流转（列表+详情）| 调度/片区主管 | 追踪事件流转节点，明确责任人、下一步动作和时限 |
| Tab 2 操作面板 | 调度/片区主管 | 推进事件流转（确认/改派/升级），形成操作反馈 |
| Tab 3 AI 风险态势 | 高管层 + 调度主管 | 全局看清风险分布和已处理进度 |
| Tab 3 AI 建议卡片 | 调度/片区主管 | 辅助判断优先级、推荐动作和资源匹配 |
| Tab 3 生成建议（v1.2.0）| 调度/片区主管 | 对无预置建议的事件实时请求 AI 判断 |
| 跨模块导航（事件详情→AI）| 调度/片区主管 | 无缝串联"看事件→看建议→做决策"流程 |
